Création d'un script
#php #composer
3-5 minutes
11 février 2023

Voici les étapes pour créer des scripts Composer pour administrer une base de données MySQL dans un projet Symfony 5.4 :
1. Créez un fichier composer.json à la racine de votre projet et définissez un script personalisé.



     swift
        {
          "scripts": {
            "post-install-cmd": [
              "YourVendor\\YourBundle\\Composer\\ScriptHandler::createSchema"
            ],
            "post-update-cmd": [
              "YourVendor\\YourBundle\\Composer\\ScriptHandler::updateSchema"
            ]
          }
        }
          

      sql
      php bin/console doctrine:schema:update --force
Créez un répertoire Composer dans le bundle de votre choix (par exemple, YourVendor\YourBundle\Composer) et créez un fichier ScriptHandler.php dans ce répertoire.
Dans le fichier ScriptHandler.php, définissez les méthodes createSchema et updateSchema qui vont être appelées par les scripts définis dans le fichier composer.json.



      php
      < ?php

      namespace YourVendor\YourBundle\Composer;

      use Symfony\Component\Process\Process;

      class ScriptHandler
      {
        public static function createSchema()
        {
          $process = new Process('php bin/console doctrine:schema:create');
          $process->run();
          if (!$process->isSuccessful()) {
            throw new \RuntimeException(sprintf(
            'An error occurred while creating the database schema: %s',
            $process->getErrorOutput()
            ));
          }
        }

        public static function updateSchema()
          {
            $process = new Process('php bin/console doctrine:schema:update --force');
            $process->run();
            if (!$process->isSuccessful()) {
              throw new \RuntimeException(sprintf(
              'An error occurred while updating the database schema: %s',
              $process->getErrorOutput()
              ));
            }
          }
        }

Exécutez les scripts en lançant la commande suivante :

     composer install
ou

     composer update

Ces étapes vous permettront de créer des scripts Composer pour administrer une base de données MySQL dans un projet Symfony 5.4.
Vous pouvez personnaliser les scripts en ajoutant d'autres commandes de gestion de base de données pour répondre à vos besoins spécifiques.
